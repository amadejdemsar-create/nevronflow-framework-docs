Architecture

Four layers of capability. Claude Code reads the orchestrator instructions, delegates work to surface agents, invokes skills as slash commands, and is stopped by the hook whenever it attempts to skip delegation.

The five feedback loops

Every non-trivial change in nevron-core flows through one or more of these loops. Skipping steps is the main failure mode the framework exists to prevent.

01
Clarify → Plan → Implement → Verify → Explain
Every non-trivial task flows through all five. Skipping any is the main failure mode the workflow rule prevents.
02
Diagnose before fix
/nf:diagnose enumerates at least 5 candidate causes with evidence before any code change proposal.
03
Delegate before edit
The PreToolUse hook blocks direct source edits. Every backend / frontend / PWA / TV change goes through the matching surface agent.
04
Migrate with approval
/nf:db-change shows SQL and asks before any schema mutation. Never autoruns migrate.
05
Analytics by default
Every new guest / revenue / operational feature must include a metric decision in its plan.

The four-layer pattern

Every enforced behavior follows the same shape. Doctrine explains WHY, the rule fires automatically, the agent makes judgment calls, the skill mechanizes the boilerplate.

Step 1
Doctrine
docs/<TOPIC>_GUIDE.md. Explains the principles and patterns. Read once per contributor.
Step 2
Rule
.claude/rules/<topic>.md. Fires automatically at plan or decision time. Blocks silent skipping.
Step 3
Agent
.claude/agents/<topic>-*.md. Read-only judgment. Proposes decisions without implementing.
Step 4
Skill
.claude/skills/nf-<topic>/SKILL.md. Mechanizes the boilerplate. Delegates to surface agents.

Example: analytics. Doctrine = ANALYTICS_CONTRIBUTION_GUIDE.md, rule = analytics-for-new-features.md, agent = nevron-analytics-architect, skill = /nf:add-analytics.

How it works

Claude is the orchestrator, not the implementer. It reads your request, decides what to delegate, invokes skills as slash commands, and is stopped by the hook whenever it tries to edit source code without going through a surface agent.

Who does what

Three actors: you, Claude (the orchestrator), and the agents. The hook is not an actor but a mechanical gate. Each has a clear lane.

Claude (the orchestrator)

Acts alone because of CLAUDE.md
  • Reads your request and asks clarifying questions if vague
  • Decides which agent(s) or skill(s) to invoke
  • Writes agent prompts with clear context and constraints
  • Synthesizes results from multiple agents
  • Runs verification skills after implementation (/nf:test, /nf:review, /nf:verify-ui)
  • Translates technical changes into business language
  • Manages session state (/nf:save, /nf:continue)
  • Edits trivial config files, CLAUDE.md, docs, .claude/ configs directly
  • Reads and navigates the codebase freely (Grep, Glob, Read)

Agents (the implementers)

Act only when Claude delegates
  • Modify source files in their specific surface (backend / frontend / pwa / tv / pms / widget)
  • Follow the patterns defined in their rule files
  • Run builds and tests within their surface
  • Return a report of what changed and what to verify
  • Read-only agents (feature-planner, diagnostician, architect, checkers) never edit anything
  • Never decide scope on their own; scope is set by the orchestrator

The hook (enforce-delegation.sh)

Blocks illegal actions at the tool-call level
  • Fires on PreToolUse for Edit, Write, MultiEdit, NotebookEdit
  • Checks: is the target path under backend/, frontend/, pwa/, tv/, pms-hub/, or widget/?
  • If YES and the call is from the main session (no agent_id), it BLOCKS and forces delegation
  • If the call is from a subagent, it passes through
  • One-shot escape for truly trivial edits: touch .claude/ALLOW_DIRECT_EDIT (auto-deletes after use)

The decision tree

Claude runs this decision tree on every user request. It's encoded in CLAUDE.md and the nevronflow-workflow rule, reinforced by auto-memory.

USER REQUEST
  │
  ├─ Is the request vague or has multiple interpretations?
  │     YES → ask clarifying questions first. STOP.
  │     NO  → continue.
  │
  ├─ Is the user reporting a bug / symptom / "it says X"?
  │     YES → invoke /nf:diagnose
  │             └─ delegates to nevron-diagnostician
  │             └─ returns ranked report with 5+ candidate causes
  │             └─ no fix proposal until report returns
  │     NO  → continue.
  │
  ├─ Is it a new feature or non-trivial change?
  │     YES → invoke /nf:plan or feature-planner agent
  │             ├─ Large feature?    use /nf:plan (4 phases: discovery → research → PRD → prompts)
  │             ├─ Medium?           use feature-planner directly
  │             └─ Guest/revenue/ops? ALSO invoke nevron-analytics-architect during planning
  │           Present plan → WAIT for user approval before implementing.
  │     NO  → continue.
  │
  ├─ Does the plan involve a database change?
  │     YES → invoke /nf:db-change
  │             └─ delegates to migration-planner
  │             └─ shows SQL, assesses risk, never auto-runs migrate
  │     NO  → continue.
  │
  ├─ Does implementation require editing files under backend/, frontend/, pwa/, tv/, pms-hub/, widget/?
  │     YES → delegate to the matching surface agent:
  │             nevron-backend    for backend/
  │             nevron-frontend   for frontend/
  │             nevron-pwa        for pwa/
  │             nevron-tv         for tv/
  │             nevron-pms        for pms-hub/
  │             nevron-frontend   for widget/
  │           If cross-surface: run agents in parallel.
  │           ⚠ HOOK fires here if you skip delegation
  │     NO  → Claude edits directly (docs, configs, CLAUDE.md, .claude/ meta).
  │
  ├─ After implementation: run verification
  │     Backend changes    → /nf:test
  │     Frontend/PWA       → /nf:verify-ui (delegates to ui-verifier)
  │     Any change         → /nf:review (delegates to code-reviewer + tenant-safety-checker)
  │     Before merge       → /nf:deploy-check
  │
  └─ Explain to the user in business language
        (what changed, what to test, before vs after)

Skills → agents wiring

Every skill that delegates work shows which agents it spawns. Skills without agent delegations are either session-state helpers (/nf:save, /nf:continue) or direct command runners (/nf:test, /nf:maestro).

/nf:db-change migration-planner Shows SQL, downtime risk, safe migration path. Never auto-runs migrate.
/nf:review code-reviewer tenant-safety-checker Full review + tenant safety audit on recent changes.
/nf:tenant-check tenant-safety-checker Quick isolation audit on backend files changed since last commit.
/nf:verify-ui ui-verifier Playwright-driven verification of UI changes. Screenshots included.
/nf:deploy-check tenant-safety-checker ui-verifier Pre-merge validation: tests, lint, types, migrations, secrets, Docker, tenant safety.
/nf:add-analytics nevron-analytics-architect nevron-backend nevron-frontend ui-verifier Architect already ran (gives metrics). Backend+frontend scaffold boilerplate. Verifier tests.
/nf:push-template nevron-push nevron-backend nevron-frontend Scaffolds a push notification template across backend and admin UI.
/nf:hotel-onboard nevron-hotel-onboarder Walks hotel GM through Apple Developer enrolment, D-U-N-S, signing setup.
/nf:mobile-setup nevron-mobile nevron-codemagic nevron-app-store nevron-hotel-onboarder nevron-push nevron-pms Initial mobile infra setup. Touches everything: native, CI, store, onboarding, push, PMS.

/nf:codemagic-status none (direct API call) CI/CD pipeline status summary.
/nf:test none (runs pytest directly) Smart test runner; detects changed files, runs only affected tests.
/nf:maestro none (runs Maestro CLI) E2E tests for given surface.

/nf:continue none (reads .nevronflow/) Resume from saved state.

Three common flows

Worked examples showing the full dance between user, Claude, agents, skills, and the hook.

Flow 1 · Bug report
"The catalogue filter isn't showing categories when I pick a module."
01
User
Reports the bug in plain language.
02
Claude
Recognizes this is a bug report (not a feature request). Per the decision tree, invokes /nf:diagnose BEFORE proposing any fix.
Forbidden: "it's probably X, let me fix it." The workflow rule and auto-memory both enforce diagnosis first.
03
/nf:diagnose
Delegates to nevron-diagnostician.
04
nevron-diagnostician (read-only)
Enumerates 5+ candidate causes. For each: grep-matches the user-visible string, tests the hypothesis against code, rules it in or out with evidence. Returns a ranked report.
Example: 1) selector uses wrong field name (LIKELY) 2) interceptor strips query param (unlikely) 3) backend filter logic broken (unlikely) 4) stale cache (ruled out, cache TTL is 0) 5) frontend state not updating (possible, check signal).
05
Claude
Presents the diagnosis. Proposes the top-ranked fix. Waits for user approval.
06
User
"Yes, fix it."
07
Claude
Attempts to edit backend/ebiplatform/catalogues/selectors.py directly.
08
Hook: enforce-delegation.sh
BLOCKS. "Main session attempted to Edit under backend/. Delegate to nevron-backend."
09
Claude
Delegates to nevron-backend agent with the diagnosis, the affected file, and the exact change.
10
nevron-backend
Makes the fix following HackSoft patterns. Runs affected tests. Returns diff + test results.
11
Claude
Invokes /nf:test to confirm broader test suite passes, then /nf:verify-ui to confirm the UI now behaves correctly.
12
Claude
Reports to user in business language: "Before: clicking a module showed zero categories. After: categories appear correctly. Test by going to Catalogues and selecting any module."
Flow 2 · New feature with analytics
"We need a Spa module where guests can book treatments."
01
Claude
This is a feature request. Invokes /nf:plan.
02
/nf:plan
Phase 1: Discovery questions. Asks user about pricing, target hotels, integration with PMS, treatment catalogue structure, deposit handling, cancellation policy. Saves answers to .nevronflow/DECISIONS.md.
03
/nf:plan
Phase 2: Cross-niche research. Invokes nevron-researcher.
04
nevron-researcher (read-only)
Studies how spas, airlines, and restaurant reservation systems handle bookings. Returns 3+ approaches with pros/cons.
05
Claude
Rule fires: analytics-for-new-features. Spa is commerce + engagement. Must include Analytics section in the plan. Invokes nevron-analytics-architect.
06
nevron-analytics-architect (read-only)
Proposes 8 metrics: total_bookings, volume_trend, status_breakdown (baseline), total_revenue, revenue_trend, top_services, avg_booking_value (commerce), + cancellation_reason (deferred V2).
07
Claude
Phase 3: Writes PRD including Analytics section. Phase 4: Writes task prompts. Presents full plan to user. Waits for approval.
08
User
"Approved. Start Task 1."
09
Claude
Task 1 involves a new DB model → invokes /nf:db-change.
10
/nf:db-change
Delegates to migration-planner. Shows SQL. Assesses risk. User approves.
11
Claude
Delegates SpaBooking model + service + selector + viewset to nevron-backend, and the admin UI to nevron-frontend (parallel).
12
Claude
After the feature's model and basic viewset exist, invokes /nf:add-analytics spa.
13
/nf:add-analytics
Reads the approved architect proposal. Delegates selector scaffolding to nevron-backend, TS interface + module card to nevron-frontend, tests to nevron-backend. Calls ui-verifier at the end.
14
Claude
Runs /nf:review and /nf:deploy-check. Explains changes in business language. Hands off for merge.
Flow 3 · UI operation (not code, not plan)
"Change this tenant's AI Assistant from Custom to PWA Assistant."
01
User
Asks Claude to change a tenant setting in the running app.
02
Claude
Rule 7 of the workflow rule kicks in: do NOT write step-by-step click instructions for the user. Claude has Playwright access via ui-operator. Delegates.
03
ui-operator
Logs in as global admin, navigates to the tenant, opens AI Assistants, edits the assistant, changes the type to PWA, saves, verifies in the list. Returns screenshots and confirmation.
04
Claude
Reports: "Changed assistant type to PWA Assistant. Screenshots attached. Verified the change persisted by reopening the edit dialog."

Exception: if the action requires physical access (MFA on phone, credit card for payment, secret not shared), Claude asks the user to do it manually. Otherwise: ui-operator drives.

Rules (16)

Doctrine and style guides. Referenced by agents, Claude, and the team at decision time.

nevronflow-workflow core
Mandatory workflow: clarify → plan → delegate → verify. The orchestration spine.
multi-tenant critical
Tenant isolation rules. A violation means Hotel A can see Hotel B's guest data. Most critical rules in the system.
backend-style
Django backend: HackSoft pattern. Services (writes), selectors (reads), thin viewsets.
frontend-style
Angular 19 admin panel: standalone components, signals, PrimeNG 19 / Aura theme, Tailwind CSS.
pwa-style
Ionic 8 guest PWA: Capacitor, offline-first, guest auth, in-stay assistant. NgModule pattern, not standalone.
tv-style
Vue 2 TV client: Tizen / LG constraints, memory limits, no modern JS. Remote-control navigation.
widget-style
Preact chat widget: Shadow DOM isolation, SSE streaming, bundle size constraints.
mobile-style
Capacitor native layer: variants, on-device tenant isolation, biometric gate, offline cache TTLs.
api-conventions
REST URL structure, pagination format, auth mechanisms, error shapes.
testing
pytest patterns, coverage targets, test organization, tenant-safety test fixtures.
hospitality-domain
Hotel industry glossary. Folio, stay, PMS, guest journey, access group translations.
ai-chat-context
Two chatbot audiences: widget (pre-stay, potential guests) vs PWA (in-stay, current guests).
business-context
Reference to BUSINESS_CONTEXT.md for pricing, module value, customer segment priorities.
app-store-compliance
Apple + Google store policies: reviewer notes, metadata, data safety, rejection response playbook.
codemagic-ci
CI/CD conventions: workflow naming, signing groups, fastlane policy, validation after every YAML edit.
analytics-for-new-features new
Every new feature plan must include an Analytics section. Fires during /nf:plan and feature-planner.

Agents (20)

Claude orchestrates; agents implement or analyze. Each has defined tools and a clear purpose.

Implementation agents (Write / Edit enabled)

These agents receive delegation from Claude and modify source files in their surface.

nevron-backend write
Django backend. 40+ ebiplatform apps, HackSoft patterns, tenant safety. For any backend feature work.
nevron-frontend write
Angular admin panel. Standalone components, signals, PrimeNG / Aura, Tailwind. For admin UI.
nevron-pwa write
Ionic guest PWA. Capacitor, offline-first, guest chat. For guest-facing features.
nevron-tv write
Vue 2 TV client. Hardware constraint awareness, memory-limited environments.
nevron-pms write
FastAPI PMS Hub. Adapters for Opera, Protel, FIAS, Clock, Bentral. Data mapping, error handling.
nevron-mobile write
Capacitor native layer. Variant workflows, device-level tenant isolation, biometric gate.
nevron-push write
Push notifications. fcm-django + firebase-admin, Android channels, cost controls.
nevron-codemagic write
CI/CD. Owns codemagic.yaml, fastlane config, variant-aware build workflows, signing certificates.
nevron-app-store write
Apple + Google submission. Reviewer notes, store metadata, screenshots, privacy, rejection responses.
nevron-hotel-onboarder write
Hotel GM onboarding. Walks non-technical GM through Apple Developer enrolment, D-U-N-S, signing.
ui-operator write
Performs tasks in the running app via Playwright. Clicks, creates records, tests flows by acting.

Read-only agents (judgment + investigation)

These agents analyze, propose, and verify without modifying any files.

feature-planner read-only
Translates business language into technical plans. Maps files, identifies canonical examples, estimates scope.
nevron-researcher read-only
Cross-industry innovation research. Studies how airlines, restaurants, cruise ships solve similar problems.
nevron-analytics-architect read-only new
Proposes which metrics matter for a new feature. Judges cost, data sources, V1/V2 split.
nevron-diagnostician read-only
Systematic bug investigation. Enumerates 5+ candidate causes with evidence, ranks ruled-out section visible.
migration-planner read-only
Plans Django migrations. Shows actual SQL, assesses downtime risk, never runs migrate.
tenant-safety-checker read-only
Audits for multi-tenant isolation violations. Checks for schema leakage, cross-tenant access, missing context.
code-reviewer read-only
Pattern compliance, security, performance, test coverage, tenant safety. Structured report with criticality.
integration-checker read-only
Verifies cross-component wiring: API contracts match frontend calls, routes resolve, data flows end-to-end.
ui-verifier read-only
Verifies UI changes in the running app via Playwright. Brainstorms test scenarios, clicks to exercise change.

Skills (23)

Mechanical procedures the user invokes with /nf:<name>. Each skill has a specific trigger and outcome.

Planning and discovery

Used at the start of any non-trivial change.

/nf:plan
4-phase planning pipeline: Discovery Questions → Cross-Niche Research → Machine-Optimized PRD → AI Implementation Prompts. Most thorough.
/nf:roadmap
AI-managed roadmap via GitHub Projects. View sprint, add items, reprioritize, plan sprints, mark done.

Diagnosis and code quality

Quality gates before and after implementation.

/nf:diagnose
Systematic bug investigation via nevron-diagnostician. Enumerates all plausible causes, ranks ruled-out ones visible.
/nf:review
Combined code review + tenant safety check. Runs code-reviewer and tenant-safety-checker agents on recent changes.
/nf:tenant-check
Quick tenant isolation audit on backend files changed since last commit. Pass/fail with line numbers.
/nf:test
Smart test runner: detects changed files, runs only affected tests, translates results to plain language.
/nf:verify-ui
Manual UI verification via Playwright. Wraps ui-verifier so the team can run verification on demand.
/nf:maestro
E2E tests across surfaces (pwa, admin, super-admin, tenant, or all).

Database and deployment

Gate-controlled mutations and releases.

/nf:db-change
DB change safety protocol. Launches migration-planner. Shows SQL, assesses risk, never auto-runs.
/nf:deploy-check
Pre-merge validation: tests, lint, types, migration consistency, secrets scan, Docker build, tenant safety.

Mobile app lifecycle

Capacitor native builds, variants, store submissions.

/nf:mobile-setup
Initial mobile infrastructure setup for a fresh variant or new project.
/nf:mobile-new-variant
Create a new branded hotel variant with config, signing, and CI hooks.
/nf:mobile-build
Build mobile app(s) for a given variant via Codemagic.
/nf:mobile-verify
Verify mobile build on a physical device, including deep link handling.
/nf:mobile-submit
Pre-submission checklist and submission to App Store and Google Play.
/nf:codemagic-status
Check current CI/CD pipeline status across workflows.

Content and communication

User-facing text, translations, explanations.

/nf:translate
Add a translatable string across all locale files (Django .po + Angular i18n). Marks for review.
/nf:explain
Explain code in business language. Ties what code DOES to hotel operations, not how it works technically.
/nf:push-template
Push notification template scaffolding.
/nf:hotel-onboard
Hotel GM onboarding assistance. Walks through Apple Developer enrolment, D-U-N-S, signing.

Analytics

Part of the new analytics framework.

/nf:add-analytics new
Scaffolds analytics boilerplate (selector, TS interface, module card, tests) after architect's proposal is approved.

Session state

Cross-session persistence.

/nf:save
Save current session state to .nevronflow/PROGRESS.md. Any team member can resume with /nf:continue.
/nf:continue
Resume from saved state. Reads PROGRESS.md, PLAN.md, PRD.md to restore context.

Hooks (1)

Automatic enforcement at the tool-call level. Runs before Claude can execute certain actions.

enforce-delegation.sh blocker
PreToolUse hook. Blocks Edit / Write / MultiEdit / NotebookEdit from the main session when the target is under backend/, frontend/, pwa/, tv/, pms-hub/, or widget/. Subagent-originated edits pass through automatically. One-shot escape: touch .claude/ALLOW_DIRECT_EDIT.
